# Top Processes にメモリ順ソートと %MEM 表示を追加

## 背景

- ホストのメモリ使用率が徐々に上がってきており、 どのプロセスが占有しているかを画面上から素早く特定したい
- 現状の Top Processes カードは CPU 使用率順の固定ソートで、 RSS
  は列としては見えるがランキング上位に「メモリを最も食っているプロセス」が並ぶとは限らない
- `top` のように、 用途に応じて CPU 順 / メモリ順を切り替えて見たい

## 現状の挙動

- Top Processes カードは常に CPU 使用率降順で 10 件を表示 (同率時は RSS 降順で 2 次ソート)
- 列は PID / Name / CPU% / RSS の 4 列
- サーバー側 (`Collector#topProcs`) が全プロセスから CPU 順トップ 10 を抽出し、 スナップショットに含めて配信
- クライアントは受信配列をそのまま表 (`#procs tbody`) にレンダーしている

## 変更後の挙動

- **ソート切替トグルを追加**
  - Top Processes カードの見出し右端にセグメントコントロール `CPU / MEM` を配置
  - 既定は `CPU` (従来どおり)。 `MEM` を選ぶと表全体がメモリ (RSS) 降順で並び替わる
  - トグルの選択状態は `localStorage` に保存し、 リロード後も維持
  - 切替時のちらつきを避けるため、 サーバーは CPU 上位 / メモリ上位 の両集合を返し、
    クライアントは選択中の集合を表示する (常に 10 件)
- **%MEM 列を追加**
  - 列構成を PID / Name / CPU% / RSS / %MEM の 5 列に変更
  - `%MEM = プロセス RSS / MemTotal * 100` を小数第 1 位まで表示
  - モバイル幅では従来と同じくカード内で横スクロール
- **表示件数は従来どおり 10 件のまま**

## スコープ外

- プロセスに対する操作 (kill、 nice 変更など。 読み取り専用のまま)
- コマンドライン引数や CPU 時間累計など追加情報の表示
- スワップ使用量順など他のソート軸
- コンテナカードなど他カードのソート挙動 (今回は Top Processes カードのみ)
- 履歴 (グラフ) への影響 (Top Processes は現状もリアルタイムのみで履歴対象外)

## E2E テストチェックリスト

- ### 正常系
  - [ ] 初回アクセス時、 Top Processes カードに `CPU / MEM` トグルが表示され、 `CPU` が選択されている
  - [ ] `CPU` 選択時、 従来どおり CPU% 降順で 10 件表示される
  - [ ] `MEM` を押すと即座に表全体が RSS 降順に並び替わり、 上位に RSS の大きいプロセスが来る
  - [ ] 5 列 (PID / Name / CPU% / RSS / %MEM) が表示され、 %MEM の合計が概ねゲージの Memory% と整合する
  - [ ] トグル選択後にリロードしても、 前回選んだソート (`CPU` or `MEM`) が復元される
- ### 異常系
  - [ ] SSE 切断中もトグル操作は UI 上で切り替えられる (最後に受信した集合に対して並び替わる)
  - [ ] MemTotal が 0 のような異常値でも %MEM 列が `NaN` や `Infinity` にならず、 `0.0%` または `—` を表示する
- ### エッジケース
  - [ ] 起動直後、 CPU% がまだ 0 のサンプルでも `MEM` に切り替えれば RSS 降順で有意な並びが得られる
  - [ ] 短命プロセスが走査中に消えても表全体がクラッシュしない (従来どおり無視して次のスナップショットで復帰)
  - [ ] モバイル幅 (375px 相当) でもカード高さは 10 行分固定のまま、 横スクロールで %MEM 列まで到達できる

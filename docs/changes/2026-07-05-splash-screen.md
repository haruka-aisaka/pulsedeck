# スプラッシュスクリーンの実装

## 背景

PWA / ブラウザどちらでも、起動直後に空のカード（— 表示）が見えてしまい没入感に欠ける。

## 現状の挙動

ページを開くと即ダッシュボードの骨組みが表示され、1〜2 秒後にデータが流れ込む。

## 変更後の挙動

- ページを開くと全面スプラッシュ（中央に大きめのロゴ、パルス線が描画されるループアニメーション、 ワードマーク
  "PulseDeck"）が表示される
- 最初の SSE スナップショットを受信したらフェードアウトし、データが揃った状態のダッシュボードが現れる
- 一瞬で消えるとチラついて見えるため、最低 900ms は表示を維持する
- サーバーに接続できない場合でも最大 4 秒でスプラッシュを解除し、切断インジケーターが見える状態にする
- スプラッシュの配色はテーマ（ライト/ダーク）に追従する

## スコープ外

- iOS ネイティブのランチ画像（apple-touch-startup-image。端末サイズ別 PNG が大量に必要なため見送り。 Android
  は manifest から自動生成されるものをそのまま利用）

## E2E テストチェックリスト

- ### 正常系
  - [ ] リロード直後にスプラッシュが表示され、データ受信後にフェードアウトする
  - [ ] フェード後のダッシュボードは値が入った状態になっている
- ### 異常系
  - [ ] サーバー停止中でも約 4 秒後にスプラッシュが消えて切断表示が見える
- ### エッジケース
  - [ ] ライトテーマ（OS 設定 light）でもスプラッシュの配色が破綻しない

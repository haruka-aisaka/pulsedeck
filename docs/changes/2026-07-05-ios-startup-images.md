# iOS ランチ画像 (apple-touch-startup-image) の追加

## 背景

iOS の PWA はブラウザエンジン起動前に OS がランチスクリーンを描画するが、 画像未指定のため真っ白になり、edge
to edge の没入感を損なっている。

## 現状の挙動

ホーム画面から起動すると、白い画面 → in-app スプラッシュ → ダッシュボードの順に表示される。

## 変更後の挙動

- 主要な iPhone（10 サイズ・縦向き）と iPad（5 サイズ・縦横両向き）のランチ画像を用意し、 media query 付きの
  `apple-touch-startup-image` として登録する
- 起動シーケンスが「OS ランチ画像（ダーク背景 + 中央ロゴ）→ in-app スプラッシュ → ダッシュボード」と
  白を挟まず連続する
- 画像は in-app スプラッシュと同デザイン（ダーク固定。iOS のランチ画像はテーマ追従不可）
- リストにない画面サイズの端末では従来どおり白（動作は壊れない）

## スコープ外

- ライトテーマ用ランチ画像（prefers-color-scheme 付き media query の端末側サポートが不安定なため）
- Android（manifest から自動生成されるものを利用、対応済み）

## E2E テストチェックリスト

- ### 正常系
  - [ ] 全サイズの PNG が生成され、HTTPS 経由で 200 / image/png で配信される
  - [ ] link タグの media query と画像のピクセルサイズが一致している
- ### 異常系
  - [ ] 画像が増えてもページ本体の読み込み・SW キャッシュに影響がない
- ### エッジケース
  - [ ] リスト外の画面サイズでもページは正常に動作する

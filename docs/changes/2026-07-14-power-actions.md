# ホストリブートとコンテナ再起動アクションを追加

## 背景

- PulseDeck は現状 read-only ダッシュボードで、 メトリクスを見るだけの用途に留まっている
- 実運用では 「メモリを掴んでいるプロセスを見て → その特定コンテナを再起動」 や
  「更新後にホスト全体を安全に再起動」 といった、 見てから即介入したいケースが多い
- SSH や ttyd に切り替えず、 同じ画面から最小限の電源操作ができると Pi 前提の運用に合う

## 現状の挙動

- Top Processes / Containers / Services 等の情報はすべて read-only
- Docker socket は read-only マウント (`/var/run/docker.sock:...:ro`)。 REST 呼び出しは GET のみ
- サーバー側 (`server/main.ts`) は `GET /api/stream` と `GET /api/snapshot` のみ提供
- ホスト再起動は SSH または物理端末経由でしか行えない

## 変更後の挙動

### 1. ホストリブートボタン

- **配置**: ヘッダー右上、 テーマ切替ボタンの左に電源アイコンボタン (`⏻`) を新設
- **クリック**: モーダルが出て「ホストを再起動しますか？」と表示、 「再起動」 / 「キャンセル」ボタン
- **再起動を押下**: モーダル内が 10 秒カウントダウン UI に切り替わり、 「あと N 秒で再起動… キャンセル」を表示
  - カウントダウン 0 になった時点でサーバーに `POST /api/reboot` を送信
  - サーバーは `pid: host` により見えているホスト PID 1 (systemd) に `SIGRTMIN+5` を送信し `reboot.target`
    を発火させる (systemd 公式の作法。 nsenter や追加の cap は不要)
- **キャンセル**: カウントダウン中いつでも押せる。 押下でモーダルを閉じ、 リクエストは送らない
- **リブート実行後**: SSE が切断されるため接続ラベルが `disconnected — retrying` に。 ホスト起動後 SSE
  が自動再接続 (既存挙動)。 特別な UI は追加しない
- **失敗時**: モーダル内にエラー行を出し、 「閉じる」で復帰。 サーバー 5xx / 権限不足時など

### 2. コンテナ個別の再起動ボタン

- **配置**: Containers カード内、 各行の末尾に「↻」アイコンボタンを追加 (running のときのみ有効)
- **クリック**: モーダルが出て「コンテナ `<name>` を再起動しますか？」と表示、 「再起動」 / 「キャンセル」
- **再起動を押下**: 即座に `POST /api/containers/<id>/restart` を送信 (10 秒カウントダウンは無し。
  誤操作しても被害範囲が限定的なため)
- **成功時**: 通常の SSE スナップショットにより State が `restarting` → `running` に自然に更新される
- **失敗時**: モーダル内にエラー行を出し、 「閉じる」で復帰

### 3. Docker socket を読み書きに変更

- `docker-compose.yml` のマウントを `:ro` から `:rw` に変更する
- README にセキュリティ節を追記して、 これが tailnet 限定公開前提であることを再確認

### 4. 認証は追加しない

- README で既に「PulseDeck has no built-in authentication. Only expose it on networks you trust」と明記済み
- リブート/再起動も同じ前提 (tailnet 限定) に乗る
- 変更定義書レベルではこの前提を明示するが、 UI 上の警告は増やさない

## スコープ外

- シャットダウン (poweroff)、 SSH セッション操作、 systemd サービスの個別再起動
- コンテナの起動/停止/削除、 pause/unpause (今回は restart のみ)
- 操作ログの永続化 (誰がいつ何を操作したかの記録)
- 認証機構の導入
- 「更新後リブート」のような一連の作業のスクリプト化
- モバイル画面での特別なレイアウト調整 (既存モーダル同様の縦積みでカバー)

## E2E テストチェックリスト

- ### 正常系
  - #### リブート
    - [ ] ヘッダーに `⏻` アイコンボタンが表示される
    - [ ] クリックで「ホストを再起動しますか？」モーダルが開く
    - [ ] 「再起動」を押すとカウントダウン UI に切り替わり、 1 秒ごとに数字が減る
    - [ ] カウントダウン中に「キャンセル」を押すとモーダルが閉じ、 サーバーには何も送られない
    - [ ] カウントダウン 0 で `POST /api/reboot` が発火する (実行そのものはテストでは検証しない)
  - #### コンテナ再起動
    - [ ] running なコンテナ行の末尾に「↻」ボタンが出る
    - [ ] クリックで確認モーダルが出て、 コンテナ名が正しく表示される
    - [ ] 「再起動」で `POST /api/containers/<id>/restart` が発火する
    - [ ] コンテナが `running` → `restarting` → `running` と遷移して UI に反映される
- ### 異常系
  - [ ] `POST /api/reboot` がサーバー側で失敗 (5xx) した場合、 モーダル内にエラーが出て閉じる操作ができる
  - [ ] Docker socket が使えない環境で `POST /api/containers/*/restart` が 503 で返り、 UI がエラー表示する
  - [ ] 停止中コンテナ (exited など) には「↻」ボタンが出ない
- ### エッジケース
  - [ ] リブートモーダルが開いた状態で SSE 切断してもキャンセル操作は動く
  - [ ] カウントダウン中にブラウザを閉じてもクライアント側のタイマーだけで走っているため、
        サーバーへは飛ばない
  - [ ] コンテナ再起動確認モーダル中に対象コンテナが消えた場合、 送信すると 404 → エラー表示
  - [ ] モバイル幅 (375px) でも `⏻` ボタンと モーダルが操作できる

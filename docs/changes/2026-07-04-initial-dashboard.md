# PulseDeck 初期リリース: サーバー状況ダッシュボード

## 背景

Raspberry Pi サーバーの状況（CPU・メモリ・温度・ディスク・ネットワーク・Docker コンテナ・プロセス）を、
外部（Tailscale 経由）からブラウザでひと目で確認できるダッシュボードが欲しい。
OSS として公開できる品質・構成にする。

## 現状の挙動

サーバー状況を確認するには SSH でログインして `top` / `docker stats` 等を叩く必要がある。

## 変更後の挙動

- ブラウザで `https://<ホスト名>.<tailnet>.ts.net/` を開くと、ダークテーマのダッシュボードが表示される
- 表示内容（2 秒ごとに SSE で自動更新、リロード不要）:
  - ヘッダー: ホスト名・稼働時間・OS/カーネル・接続状態インジケーター
  - ゲージ: CPU 使用率・メモリ使用率・CPU 温度・ディスク使用率
  - 履歴グラフ: CPU / メモリ / 温度 / ネットワーク送受信の直近推移（サーバー側リングバッファ、最大 3 時間）
  - CPU コア別使用率バー
  - Docker コンテナ一覧: 名前・イメージ・状態・CPU%・メモリ使用量（10 秒ごと更新）
  - プロセス Top 10（CPU 順）: PID・名前・CPU%・メモリ
- 接続が切れた場合はヘッダーのインジケーターが赤くなり、自動で再接続する
- UI 言語は英語（OSS として公開するため）

## スコープ外

- 認証機能（Tailscale の tailnet 内アクセスに限定する前提。パブリック公開は想定しない）
- アラート・通知機能
- 複数サーバーの集約表示
- メトリクスの永続化（再起動で履歴は消える）

## E2E テストチェックリスト

- ### 正常系
  - [ ] トップページが表示され、全カード（ゲージ・グラフ・コア・コンテナ・プロセス）にデータが出る
  - [ ] 数秒待つと値が自動更新される（SSE）
  - [ ] Docker コンテナ一覧に稼働中コンテナ（pulsedeck 自身を含む）が表示される
  - [ ] `docker compose up -d` で起動し、再起動ポリシーが効いている
- ### 異常系
  - [ ] サーバーを止めるとインジケーターが切断表示になり、再起動すると自動復帰する
  - [ ] docker.sock がマウントされていない場合もコンテナカード以外は動作する
- ### エッジケース
  - [ ] 起動直後（履歴が空）でもグラフがエラーにならない
  - [ ] リロード後も履歴グラフに過去分（サーバー保持分）が表示される
  - [ ] 狭い画面幅（モバイル相当）でレイアウトが崩れない

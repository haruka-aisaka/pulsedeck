# サービス一覧の追加・グラフ 10 分化・自動起動・edge to edge 対応

## 背景

- サーバー内で立ち上げている Web サービスのポートマッピングを覚えられないため、URL 一覧が欲しい
- 履歴グラフは直近 10 分が見られれば十分（3 時間分の保持・描画は過剰）
- 再起動後も pulsedeck が自動で立ち上がってほしい
- モバイルで edge to edge 表示に対応したい

## 現状の挙動

- グラフは最大 3 時間分を保持・描画
- 稼働サービスの URL 一覧は存在しない
- compose に `restart: unless-stopped` はあるが、再起動後の自動起動は未検証
- ノッチ・ホームバー領域のセーフエリアは未考慮

## 変更後の挙動

- **グラフ**: 常に直近 10 分の固定ウィンドウ。履歴保持も 10 分に短縮（メモリ・転送量も削減）
- **Services カード**（新設）: ホストで LISTEN 中の TCP ポートと Docker のポート公開を自動検出し、
  クリックできる URL チップ（`http://<アクセス中のホスト名>:<ポート>`）として一覧表示。 名前は Docker
  コンテナ名 > プロセス名の順で決定。SSH(22)・DNS(53) は除外。 更新は閲覧中のみ 60 秒ごと + 接続直後
- **自動起動**: Docker サービスの systemd 有効化を確認し、`restart: unless-stopped` により再起動後に自動復帰
- **edge to edge**: `viewport-fit=cover` + セーフエリア inset 対応、`theme-color` 設定

## スコープ外

- UDP サービスの検出、コンテナ内部ポート（未公開ポート）の表示
- サービスの死活監視・ヘルスチェック
- 手動でのサービス名上書き設定

## E2E テストチェックリスト

- ### 正常系
  - [ ] Services カードに稼働中サービス（pulsedeck 自身 8480 を含む）がリンク付きで表示される
  - [ ] Docker 公開ポートはコンテナ名で表示される
  - [ ] グラフの X 軸が直近 10 分固定になる
- ### 異常系
  - [ ] LISTEN ポートが 1 つもなくてもカードがエラーにならない
- ### エッジケース
  - [ ] 127.0.0.1 のみで LISTEN しているポートは表示されない（外部から到達不能のため）
  - [ ] 22/53 番ポートは表示されない
  - [ ] docker restart 後に自動で復帰する（再起動時挙動の代替検証）
